Box[T]'s own bare, unbound type-parameter field (value: T) — permanent, not just unbuilt. I spiked whether reflection could write it generically and it's a real language wall: init_pointee_move requires T: Movable, but reflect[T].field_types() only ever hands back AnyType-bounded types, with no escape to assert a trait bound generically. Gets its own clear compile-time error now instead of a confusing one. to_json still works fine (reading needs no bound).

A container wrapper with 2+ type arguments other than Dict — no defined JSON array shape for arbitrary multi-argument container semantics, deliberately rejected.

An undiscovered generic type with exactly one type argument (e.g. an external Box[String], not Box-the-DSL-plain-struct) gets misrouted into the container escape hatch, not the new leaf one — TypeExpr classifies any Ident[args...] as PARAMETERIZED, so a single-type-arg undiscovered type is indistinguishable from a real container at that point. It'll compile only if you write the two _json_to_list/_json_from_list companions (as if it were a container), even though semantically it's a single-value box, not a collection. There's currently no way to tell the compiler "this one-type-arg external type is Box-shaped, not List-shaped."
